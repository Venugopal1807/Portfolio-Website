Case Study: Fuel Logistics Platform (Zapygo)

🧩 Problem

Traditional fuel delivery systems lack real-time tracking, security, and reliability. Manual operations lead to errors, fuel theft risks, and poor visibility in delivery lifecycle.

⚙️ System Overview

  • 3 Applications: Customer, Driver, Petrol Pump Admin
  • Shared PostgreSQL database
  • Real-time + offline-first hybrid architecture

🧠 Architecture Decision

Initially attempted a single backend for all apps but faced coupling and complexity issues. Final approach used separate backends for better scalability and maintainability.

🔐 Key Challenges

  • Secure fuel dispensing using BLE + OTP + geofencing
  • Offline-first system using SQLite/Room with sync engine
  • Real-time order lifecycle management
  • Geo-based service zoning (30km radius)
